1. Project Overview

Budget Flow is a financial management web application that helps users track income and expenses, manage budgets, monitor savings goals, and understand spending habits more clearly.

The main purpose of the system is to simplify personal finance management and give users practical insight into how they use money over time.

  • track income and expenses
  • manage budgets
  • visualize financial data
  • get quick assistance from an AI finance assistant

2. Objectives

  • Provide a simple platform for tracking finances
  • Help users make better financial decisions
  • Automate calculations and budgeting logic
  • Offer visual insights through the dashboard and reports

3. System Architecture

The project follows a multi-layer architecture:

Frontend (UI)
↓
Backend APIs (Auth + Feature services)
↓
Database (PostgreSQL)

Frontend

The frontend uses HTML, CSS, and JavaScript. It handles UI rendering, user interaction, and navigation between screens.

Backend

The backend is split into two services.

  • Auth Backend: user registration, login, password reset, Google sign-in, and JWT authentication
  • Feature Backend: budgets, transactions, savings, reports, settings, and financial operations

Database

PostgreSQL stores the application data, including users, transactions, budgets, savings data, and user settings.

5. Installation and Setup

Requirements

  • Bun
  • PostgreSQL
  • Git

Step 1: Clone the repository

git clone https://github.com/hmekpe/Budget-Flow.git
cd Budget-Flow

Step 2: Install dependencies

Install dependencies in both backend folders.

cd Budget-Flow/backend
bun install
cd ../../budget-flow-unified-backend
bun install

Step 3: Set up environment variables

Create a `.env` file in each backend.

DATABASE_URL=postgresql://user:password@localhost:5432/budget_flow
JWT_SECRET=your_secret
CLIENT_URL=http://localhost:5500

Important:

  • both backends must use the same `JWT_SECRET`
  • both backends should connect to the same PostgreSQL database

Step 4: Run the project

.\start-workflow.cmd

6. Authentication System

Budget Flow uses JWT for authentication.

  1. User registers or logs in
  2. Server generates a token
  3. Token is stored on the client
  4. Token is used for protected API requests

The auth backend also supports Google sign-in and password reset.

9. API Overview

Auth APIs

  • `POST /register`
  • `POST /login`
  • `POST /forgot-password`
  • `POST /reset-password`
  • `POST /google`
  • `GET /me`

Feature APIs

  • `GET /transactions`
  • `POST /transactions`
  • `DELETE /transactions/:id`
  • `GET /budgets/current`
  • `GET /budgets/categories`
  • `GET /dashboard/summary`
  • `GET /reports/current`

Third-Party APIs

Groq API

  • Used for the AI chatbot
  • Endpoint: POST https://api.groq.com/openai/v1/chat/completions
  • Authentication: `GROQ_API_KEY`
  • Current model: `openai/gpt-oss-20b`

Implemented in:

  • `assistant.service.js`
  • `assistant-config.service.js`

ExchangeRate-API

  • Used for currency conversion
  • Endpoint: https://open.er-api.com/v6/latest
  • No API key required
  • Updates once per day
  • Free tier rate-limited

Implemented in:

  • `exchange-rates.service.js`

10. Deployment

Budget Flow is currently deployed on Render.

Service URL
Frontend https://bf-web.onrender.com
Auth API https://bf-auth-api.onrender.com
Feature API https://bf-feature-api.onrender.com

The working hosted setup uses:

  • one frontend service
  • one auth backend service
  • one feature backend service
  • one shared PostgreSQL database
Important deployment rule
Both backend services must use the same `JWT_SECRET`, the same database, and the same Render region as the database.

11. How to Run the App Successfully

  1. Install Bun, PostgreSQL, and Git
  2. Clone the repository
  3. Install dependencies in both backend folders
  4. Create valid `.env` files for both services
  5. Use the same database and JWT secret for both backends
  6. Run the local workflow or deploy the services in the correct order

For hosted deployment, the safest order is:

  1. Create the database
  2. Deploy the auth backend
  3. Deploy the feature backend
  4. Deploy the frontend
  5. Point the frontend to the live backend URLs

12. Conclusion

Budget Flow is a complete web application for personal finance management. It combines a clear frontend, a split backend architecture, PostgreSQL data storage, and useful third-party integrations like Groq and ExchangeRate-API to provide a practical and modern budgeting experience.

With the correct setup, the project can be run locally for development or deployed on Render for production use.
